# Disclaimer and Intended Use

## AI-Generated Content

The alignment mappings produced by this pipeline are generated by a large language model (LLM).
They represent the model's interpretation of how course learning outcomes relate to program
outcomes — **not a validated or authoritative curriculum assessment**.

All outputs **must be reviewed by a qualified subject matter expert** before being used for any
official purpose, including program review, accreditation submissions, or curriculum planning
decisions.

LLM outputs are non-deterministic. Running the same pipeline twice may produce different
mappings. Results should not be treated as reproducible without pinning the model version and
recording the run parameters.

---

## Data Handling

This pipeline is designed to process curriculum documents **locally**, using a locally-hosted
LLM endpoint. No curriculum data should leave your network during normal operation.

**However:** if `LLM_BASE_URL` is not set in your `.env` file, the pipeline will default to the
OpenAI API and **send curriculum content to a third-party cloud service**. The pipeline will
warn you before this happens and require explicit confirmation.

It is your responsibility to ensure your `.env` is correctly configured before running the
pipeline on any sensitive or proprietary institutional data.

---

## Institutional IP and Sensitive Data

Course outlines, program outcomes, and curriculum documents may constitute institutional
intellectual property. Do not commit these files to a public repository.

This repository's `.gitignore` excludes `data/raw/` and `data/processed/` by default.
**Verify this is in place before running `git add` or committing any changes.**

This project is an **experimental, personally-developed tool**. It is not an official NSCC
product and is not supported by NSCC IT or any academic governance body. There is no
helpdesk, SLA, or guarantee of continued maintenance.

Use at your own risk. Test outputs carefully before relying on them for any consequential

---

## Local AI Cluster

If using the NSCC Truro Campus AI Cluster, note that the vLLM endpoint currently has **no
authentication or rate limiting**. Any user on the NSCC internal network with knowledge of
the endpoint address can submit requests to the model. Do not use the cluster to process
data that requires access controls beyond network-level isolation.

## Summary of Key Limitations

| Limitation | Implication |
|---|---|
| AI-generated mappings | Require expert review before official use |
| Non-deterministic outputs | Two runs may differ; save timestamped runs |
| No model version pinning | Cluster updates may silently change results |
| No cluster authentication | Network-level access control only |
| Unsupported tool | No guarantee of accuracy, maintenance, or support |
| Cloud fallback risk | Misconfigured `.env` can send data to OpenAI |
